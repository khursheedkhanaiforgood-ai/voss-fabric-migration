The Three-Plane Cockpit View

The key UX innovation — derived from the Google AI dialogue §4: Digital Twin Simulation Logic. Three rows, time-aligned per migration step. All planes update simultaneously as the student enters CLI commands — just like a pilot's cockpit.

🔧 Management Plane — XIQ Intent & Policy Push

What ExtremeCloud IQ (XIQ) is doing at each step: policy pushes, device templates, Fabric Connect configuration, ZTP+ re-adoption, iqagent status. Example (Step 4 — Configure ISIS): "XIQ: Configure > Network Policy > Switching > Common Settings > Enable Fabric Connect. Set Manual Area: 00.0001."

🔄 Control Plane — IS-IS / SPB / Fabric Attach (IEEE 802.1aq + RFC 6329 + IEEE 802.1Qcj)

What IS-IS and Fabric Attach are doing: adjacency formation, I-SID advertisement via LSPs, AP3000 LLDP-FA TLV exchange. Live state overlays: adjacency UP/DOWN, I-SIDs active count. Example (Step 7 — NNI port): "IS-IS hellos sent on Port 17. Adjacency: INIT → UP."

📡 Data Plane — SPB Forwarding / MAC-in-MAC (IEEE 802.1ah)

What IEEE 802.1ah (MAC-in-MAC) is doing: fabric encapsulation, I-SID service delivery, E-LAN active/not-active, traffic paths. Live state: fabric services visible, E2E connectivity. Example (Step 9 — Assign I-SIDs): "SPB fabric now carries VLANs as I-SID services. Client frames encapsulated in backbone MAC + I-SID. Zero VLAN trunking on NNI."
