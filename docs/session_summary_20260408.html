Theory Sections Added to Welcome Page

All of the following are now on the welcome page as plain-English explanations. Next session: wrap each in st.expander() so users can skip to auth.

Section 1

What Is This Migration and Why Does It Matter?

Core difference between EXOS and VOSS: in EXOS you ARE the fabric (manual trunks everywhere); in FabricEngine you configure only the edge, fabric handles everything in between automatically.

Section 2

How Does a Phone Get an IP Address Through the Fabric?

6-step data path: Wi-Fi connect → Fabric Attach LLDP-FA TLV → DHCP Discover → IS-IS routes frame → DHCP Offer from SW1 → Internet via default route. SW2 clients reach SW1 DHCP transparently via SPB.

Section 3

Why IS-IS? What is SPBM?

IS-IS operates at Layer 2 before IP is configured. RFC 6329 extends it to carry B-MAC, I-SID, nick-name. SPBM wraps customer frames in MAC-in-MAC (IEEE 802.1ah) — backbone never learns customer MACs. Edge-only configuration principle: BEBs configure services; core switches forward IS-IS + MAC-in-MAC only.

Section 4 — Added this session

What Is XIQ's Role?

XIQ = intent layer. iqagent daemon runs on each switch/AP, HTTPS tunnel to XIQ cloud. ZTP+: factory-fresh switch after OS change gets DHCP on VLAN 1, calls home, downloads device template — zero manual login. XIQ manages: IS-IS area, SPBM, NNI port templates, VLAN definitions, FA mappings, AP policies. DHCP server pools are CLI-only (XIQ FabricEngine policy doesn't expose per-VLAN DHCP config).

Section 5

The 18 Steps as One Sentence Each — The Migration Story

7-theme story rows (color-coded by theme): each theme is one sentence describing what it accomplishes at a high level — for a student who needs the forest before the trees.
